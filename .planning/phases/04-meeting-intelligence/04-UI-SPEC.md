# Phase 4 — UI Design Contract

> Visual and interaction contract for meeting intelligence UI (templates + detection). Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (PyQt6 + QSS from design tokens) |
| Preset | not applicable |
| Component library | PyQt6 native widgets with custom painting |
| Icon library | Custom QPainter drawn icons (established pattern) |
| Font | SF Pro Text, Helvetica Neue, sans-serif (from tokens) |

---

## Spacing Scale

Declared values (inherited from existing `design/tokens_*.json`):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, dropdown-to-button gap |
| sm | 8px | Compact element spacing, structured summary section gaps |
| md | 16px | Default element spacing, section padding in summary |
| lg | 24px | Section padding within settings dialog groups |
| xl | 32px | Dialog vertical separation |

Exceptions:
- Template QComboBox next to Record button: 8px (sm) horizontal gap between combo and Record button
- Structured summary section headings: 16px (md) top margin between sections, 8px (sm) bottom margin before list items
- "Re-run AI" button: same QPushButton dimensions as existing export buttons (fixed width 120px)

## Component Inventory

### 1. Template QComboBox (new, in MainWindow control bar)

**Location:** Next to the Record button in the control bar. Left of the Record button. Per D-02: compact, always visible.

**Layout:** `[Template QComboBox] [8px gap] [RecordButton] [SystemAudioToggle] [DualLevelMeter]`

**Visual spec:**
- QComboBox, fixed width 140px, height matches control bar element height (36px)
- Font: 14px, weight 400
- Background: `background.control` (#3A3A3C / #E5E5EA)
- Border: `border.default`, borderRadius sm (6px)
- Items: "General", "Team Meeting", "1:1", "Lecture", "Interview", plus any custom templates
- Default selection: "General" (preserves existing behavior per D-03)
- Drop-down list: sorted as built-in first (in order above), then custom templates alphabetically

**States:**
- Default: "General" selected, enabled
- During recording: remains enabled (user can change template mid-recording per D-01)
- After recording with transcript selected: reflects the template used, or "General" if none was set
- No transcripts: enabled (pre-selectable for next recording)

### 2. "Re-run AI" Button (new, in TranscriptViewer)

**Location:** Below the summary tab content, left-aligned within the Summary tab panel. Per D-04 and Claude's discretion for placement.

**Layout within Summary tab:**
```
[Summary content area (QTextEdit, read-only)]
[16px gap]
["Re-run AI" button] [8px gap] [Template QComboBox (secondary)]
```

The secondary Template QComboBox allows changing the template when re-running without affecting the main control bar combo.

**Visual spec:**
- QPushButton, fixed width 120px
- Same styling as existing export buttons (control bg, primary text, border default, borderRadius sm)
- Secondary QComboBox: same spec as Component 1, fixed width 140px
- Both vertically centered on the same row

**States:**
- Default (transcript with AI results): "Re-run AI" enabled
- Default (transcript without AI results): "Run AI" label, enabled
- Disabled (no transcript selected): grayed, 50% opacity
- Running: "Running AI..." label, disabled, non-interactive
- Complete: reverts to "Re-run AI", summary content updates in place

**Behavior per D-04:** Non-destructive. Keeps original results until the new AI run completes and replaces them.

### 3. Structured Summary Display (modification to TranscriptViewer Summary tab)

**Location:** Inside the existing Summary tab QTextEdit in TranscriptViewer.

**Format depends on template type:**

**General template (backward compatible):** Plain text bullet points, identical to current behavior. No visual change.

**Structured templates (Team Meeting, 1:1, Lecture, Interview):** Rendered as HTML sections in the QTextEdit:

```html
<div style="margin-top: 0px;">
  <h3 style="font-size: 14px; font-weight: 600; color: {text.primary}; margin-bottom: 8px;">
    Decisions
  </h3>
  <ul style="margin: 0; padding-left: 20px;">
    <li style="font-size: 14px; color: {text.primary}; margin-bottom: 4px;">
      Alice: Migrate to new API by end of sprint
    </li>
  </ul>
</div>
<div style="margin-top: 16px;">
  <h3 style="...">Action Items</h3>
  <ul>...</ul>
</div>
```

**Section labels by template (per D-05):**

| Template | Section 1 | Section 2 | Section 3 |
|----------|-----------|-----------|-----------|
| Team Meeting | Decisions | Action Items | Next Steps |
| 1:1 | Discussion Points | Agreements | Follow-ups |
| Lecture | Key Concepts | Q&A | Takeaways |
| Interview | Candidate Responses | Assessment | Notes |

**Speaker attribution (per D-06):** When diarization data exists, list items include speaker names as prefixes (e.g., "Alice: Review PR by Friday"). When absent, items have no speaker prefix.

**Backward compatibility:** Display code uses `isinstance(summary, dict)` check. If `str`, renders as plain text. If `dict`, renders structured HTML sections.

### 4. Meeting Detection Notification (new, system-level)

**Mechanism:** `QSystemTrayIcon.showMessage()` per RESEARCH.md recommendation.

**Visual spec (OS-managed rendering):**
- Title: "Meeting Detected"
- Body: "{app_name} is active. Click to start recording."
- Icon: Information icon (QSystemTrayIcon.MessageIcon.Information)
- Duration: 10000ms (10 seconds)

**Interaction:**
- Click notification: opens MainWindow, pre-selects suggested template (per D-17), focuses Record button
- Dismiss notification: snoozes that app session (per D-16)
- No interaction (timeout): treated as dismiss, snoozes that session

**Template auto-suggestion (per D-17):**
| Detected App | Suggested Template |
|--------------|--------------------|
| Zoom | Team Meeting |
| Microsoft Teams | Team Meeting |
| Google Meet (Chrome) | Team Meeting |
| FaceTime | 1:1 |
| Webex | Team Meeting |
| Slack | Team Meeting |

### 5. Detection Settings Section (new, in Settings Dialog)

**Location:** New section in Settings Dialog, below existing sections. Title: "Meeting Detection".

**Visual spec:**
- Section heading: 14px, weight 600, "Meeting Detection"
- Toggle row:
  - Label (14px): "Auto-detect meetings"
  - SystemAudioToggle-style on/off switch (reuse toggle widget from Phase 2), width 44px, height 24px
  - Default: ON (per D-20)
- Help text (11px, `text.tertiary`): "Detect Zoom, Teams, Meet, and FaceTime to prompt recording."
- Spacing: 8px between toggle row and help text, 24px (lg) section padding

**States:**
- Toggle ON: detection active, help text shows as above
- Toggle OFF: detection disabled, help text changes to: "Meeting detection is off."

### 6. Template Management Section (new, in Settings Dialog)

**Location:** New section in Settings Dialog, below "Meeting Detection". Title: "Meeting Templates".

**Visual spec:**
- Section heading: 14px, weight 600, "Meeting Templates"
- Default template row:
  - Label (14px): "Default template"
  - QComboBox (same spec as Component 1, 140px width)
  - Lists all available templates, default "General"
- Templates directory row:
  - Label (14px): "Templates folder"
  - QLabel showing path: `~/.meeting_transcriber/templates/` (11px, `text.tertiary`)
  - "Open Folder" QPushButton (caption size, 100px wide) -- opens Finder at templates directory
- Help text (11px, `text.tertiary`): "Add custom templates by placing YAML files in the templates folder."

### 7. Tray Menu Detection Indicator (modification to TrayIcon)

**Location:** Existing tray icon context menu.

**Visual spec:**
- New menu item: "Meeting Detection: On" / "Meeting Detection: Off" (checkable)
- Separator above this item
- Toggle action: same as Settings toggle (synced)

### 8. Status Bar AI Progress (existing pattern, extended)

**Location:** Existing QStatusBar at bottom of MainWindow.

**Visual spec:**
- Uses existing styling patterns from Phase 3
- "Re-running AI with {template_name} template..." in `status.processing` color
- Completion: "{template_name} summary generated" in `status.success` color (3 seconds then clears)
- Error: "AI summary failed: {error}" in `status.error` color (persists until user action)

## Error and Degradation States

| Scenario | Visual Response | Copy |
|----------|----------------|------|
| AI run fails (API error) | Status bar error (persists), summary tab shows previous results or empty state | "AI summary failed: {error}. Check your API key in Settings and try again." |
| AI returns malformed JSON | Status bar warning, summary tab shows raw text fallback | "Could not parse structured summary. Showing raw output." |
| No templates directory | TemplateManager creates directory and installs built-ins on first access | No user-visible error |
| Invalid custom YAML template | Template skipped silently during load, not shown in dropdown | No user-visible error (logged) |
| Detection polling fails (NSWorkspace error) | Detection disabled silently, tray menu shows "Meeting Detection: Off" | No notification to user (logged) |
| Chrome detected but no mic activity | No notification fires (audio heuristic filters false positive per D-14) | -- |
| Notification clicked but MainWindow already recording | Notification has no effect (recording already in progress) | -- |
| No AI provider configured | "Re-run AI" button enabled but clicking shows error | "No AI provider configured. Add an API key in Settings." |
| v1.0 transcript with string summary loaded | Summary tab shows plain text (backward compatible) | No special messaging |
| v2.0 transcript with dict summary loaded | Summary tab renders structured HTML sections | No special messaging |

## Design Token Additions

No new design tokens needed. This phase reuses existing tokens:
