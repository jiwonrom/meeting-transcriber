    **2. Add CoreML conversion method to DiarizationWorker** in `src/meeting_transcriber/core/diarizer.py`:

    New private method `_try_coreml_pipeline(self, pipeline: Any) -> Any | None`:
    - Purpose: Attempt to convert pyannote segmentation model to CoreML. Return modified pipeline on success, None on failure.
    - Implementation:
      1. Try `import coremltools as ct` -- if ImportError, return None
      2. Check cached CoreML model: `coreml_path = DIARIZATION_COREML_DIR / "segmentation.mlpackage"`
      3. If cached model exists:
         a. Try to load: `import coremltools as ct; mlmodel = ct.models.MLModel(str(coreml_path))`
         b. If load succeeds, return the original pipeline (we'll set a flag indicating CoreML is available)
         c. If load fails, delete cached model and continue to conversion
      4. If no cached model, attempt conversion:
         a. `self.progress.emit("Attempting CoreML optimization...")`
         b. Extract the segmentation model from the pipeline: `seg_model = pipeline._segmentation.model`
         c. Create dummy input matching segmentation model input shape
         d. `traced = torch.jit.trace(seg_model, dummy_input)`
         e. `mlmodel = ct.convert(traced, inputs=[ct.TensorType(shape=dummy_input.shape)])`
         f. Save: `coreml_path.parent.mkdir(parents=True, exist_ok=True); mlmodel.save(str(coreml_path))`
         g. Return pipeline (conversion succeeded)
      5. On ANY exception during steps 4a-4f: log warning, return None (fallback to CPU)
    - Key: This method MUST NOT raise. All exceptions are caught and result in None return.

    **3. Modify DiarizationWorker.run()** to use CoreML gate:

    After loading pipeline and before running diarization:
    ```python
    # Attempt CoreML optimization (D-12)
    coreml_result = self._try_coreml_pipeline(pipeline)
    if coreml_result is not None:
        self.progress.emit("Identifying speakers (CoreML)...")
        # CoreML-optimized path -- pipeline internals may use ANE
    else:
        # CPU fallback -- already set via pipeline.to(torch.device("cpu"))
        self.progress.emit("Identifying speakers...")
    ```

    The actual diarization call `pipeline(str(self._audio_path))` remains the same regardless -- the CoreML optimization affects the internal segmentation model's execution device.

    **Important notes per research:**
    - CoreML conversion is HIGH RISK. The pyannote segmentation model has dynamic graph operations that coremltools may not handle.
    - The conversion attempt is a best-effort stretch goal. The CPU path is the reliable fallback.
    - D-13 (MPS) is superseded by this approach. There is no MPS fallback -- CPU is the fallback.
    - If conversion fails at runtime, the error is silently caught and CPU is used. No user-visible degradation.
