    **4. Create `src/meeting_transcriber/core/diarizer.py`** with:

    Module docstring: `"""화자 분리 — pyannote.audio 파이프라인 실행 및 세그먼트 정렬."""`

    `align_speakers(diarization_turns: list[dict], segments: list[dict]) -> list[dict]`:
    - Takes `diarization_turns` as list of `{"start": float, "end": float, "speaker": str}` dicts
    - For each segment, find speaker turn with maximum temporal overlap: `overlap = max(0, min(seg_end, turn_end) - max(seg_start, turn_start))`
    - Set `seg["speaker"] = best_speaker or ""`
    - Return modified segments list

    `rename_speaker(transcript: dict, old_label: str, new_name: str) -> dict`:
    - Update `transcript["metadata"]["speakers"][old_label] = new_name`
    - For each segment where `seg.get("speaker") == old_label`, set `seg["speaker"] = new_name`
    - Return modified transcript

    `class DiarizationWorker(QThread)`:
    - Signals: `progress = pyqtSignal(str)`, `finished = pyqtSignal(object)`
    - `__init__(self, audio_path: pathlib.Path, segments: list[dict[str, Any]], hf_token: str, parent: Any = None)`
    - Store `_audio_path`, `_segments`, `_hf_token` as private attrs
    - `run()` method:
      1. Emit `progress("Identifying speakers...")`
      2. Lazy import: `from pyannote.audio import Pipeline` and `import torch`
      3. Load pipeline: `Pipeline.from_pretrained(DIARIZATION_MODEL, token=self._hf_token)`
      4. Set device: `pipeline.to(torch.device(DIARIZATION_DEVICE))` (CPU per research -- MPS has sparse tensor bugs)
      5. Run: `diarization = pipeline(str(self._audio_path))`
      6. Extract turns: iterate `diarization.exclusive_speaker_diarization` -> list of `{"start": turn.start, "end": turn.end, "speaker": speaker}`
      7. Call `align_speakers(turns, self._segments)`
      8. Emit `finished(labeled_segments)`
      9. On any Exception `e`: emit `finished(e)`

    **5. Create `tests/test_diarizer.py`** with tests per behavior block above. Mock `pyannote.audio.Pipeline` and `torch` to avoid real model loading. Use `unittest.mock.patch` on `sys.modules` for lazy import mocking (same pattern as Phase 2 CoreAudio tests).

    Per D-04: DiarizationWorker is non-blocking QThread. Per D-12/D-13: use CPU device (MPS fallback deferred due to sparse tensor bug). Per research pitfall 4: all pyannote imports are lazy inside run() only.
