<objective>
Wire per-task AI provider selection so user's task-level overrides from `ai.task_overrides` settings are applied at runtime. Three call sites in main_window.py currently use `get_provider_chain()` (global default) instead of `get_provider_for_task()` (per-task override). Fix by changing AITaskWorker to accept ProviderManager + settings, then resolve per-task FallbackProvider inside run().

Purpose: Complete BYOK-03 requirement -- users who configure per-task provider overrides in settings get those overrides applied.
Output: Modified tasks.py, main_window.py, and updated tests.
</objective>

    def _get_provider(self, task: str) -> FallbackProvider:

        from meeting_transcriber.ai.provider_manager import FallbackProvider
