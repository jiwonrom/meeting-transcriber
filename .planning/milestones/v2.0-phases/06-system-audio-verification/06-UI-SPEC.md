# Phase 6 -- UI Design Contract

> Visual and interaction contract for Phase 6: System Audio Completion & Verification.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.

## Phase Context

Phase 6 is a **gap closure and verification phase**. All UI widgets for system audio were implemented in Phase 2 and are functional. This contract documents the existing visual contracts for the four SYSAUD requirements so that the planner and executor can verify (not re-implement) them.

**No new UI components are expected.** If verification reveals visual defects, this contract defines the correct state to fix toward.

## Component Inventory (Existing -- Verify Only)

### SystemAudioToggle (`ui/widgets/toggle_switch.py`)

| Property | Value |
|----------|-------|
| Size | 44x24px fixed |
| Thumb | 20x20px circle, 2px inset |
| Animation | 150ms InOutQuad ease |
| Track OFF | #3A3A3C (dark) / #E5E5EA (light) |
| Track ON | #FF453A (dark) / #FF3B30 (light) -- accent/recording color |
| Disabled state | 50% alpha track, muted thumb |
| Cursor (enabled) | PointingHandCursor |
| Cursor (disabled/recording) | ForbiddenCursor |

States to verify:
1. **Idle, BlackHole available, OFF**: Track gray, thumb left, PointingHandCursor
2. **Idle, BlackHole available, ON**: Track red/accent, thumb right, PointingHandCursor
3. **BlackHole not installed**: ForbiddenCursor, click emits `setup_requested`
4. **Recording active**: ForbiddenCursor, click ignored

Tooltip copy:
- Default: "Capture system audio alongside microphone"
- BlackHole missing: "BlackHole audio driver required -- click to set up"
- Recording: "Cannot change audio source while recording"

## Interaction Contracts (Verify)

### SYSAUD-01: BlackHole Detection Visual Feedback

| Trigger | Visual Response |
|---------|-----------------|
| App launch, BlackHole installed | Toggle enabled, PointingHandCursor |
| App launch, BlackHole not installed | Toggle appears disabled-style, ForbiddenCursor |
| Settings dialog opened | "BlackHole Status" section shows installed/not installed with success/error color |

### SYSAUD-02: Setup Wizard Flow

| Trigger | Visual Response |
|---------|-----------------|
| Click toggle when BlackHole missing | `setup_requested` signal fires, wizard dialog opens |
| Wizard step navigation | Step indicator updates ("Step N of 5"), heading changes |
| BlackHole detected mid-wizard | Detection polling shows success status |
| Aggregate Device created | Wizard emits `setup_completed`, toggle auto-checks ON |

### SYSAUD-03: System Audio Source Selection

| Trigger | Visual Response |
|---------|-----------------|
| Toggle ON (BlackHole available) | Track animates to accent red (150ms), thumb slides right |
| Toggle OFF | Track animates to gray, thumb slides left |
| Setting persists | `audio.system_audio.enabled` saved to settings.json |
| Start recording with toggle ON | Aggregate Device resolved, dual level meter activates |

### SYSAUD-04: Dual-Channel Visual Feedback

| Trigger | Visual Response |
|---------|-----------------|
| Recording starts, system audio ON | DualLevelMeter switches to dual mode (both bars + labels visible) |
| Recording starts, system audio OFF | DualLevelMeter stays in single mode (mic bar only) |
| Mid-recording system audio failure | Falls back to mic-only, status bar shows fallback message |
| Recording stops | Both bars reset to 0 |
