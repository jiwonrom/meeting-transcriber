    **Write tests first (RED), then implement (GREEN).**

    Add to tests/test_exporter.py:
    - Import new functions: export_to_srt, export_to_vtt, export_to_obsidian, obsidian_filename
    - Use existing _sample_transcript() fixture (has segments with start/end/text)
    - Add a _sample_transcript_with_speakers() that adds "speaker" field to segments
    - Write all tests from the behavior block above

    Then add to src/meeting_transcriber/storage/exporter.py:

    1. `_format_srt_timestamp(seconds: float) -> str` — per D-01:
       - Convert seconds to `HH:MM:SS,mmm` format (comma for milliseconds)
       - `total_ms = int(seconds * 1000)`, then extract h/m/s/ms
       - Return `f"{h:02d}:{m:02d}:{s:02d},{ms:03d}"`

    2. `_format_vtt_timestamp(seconds: float) -> str` — per D-01:
       - Same logic but period instead of comma: `f"{h:02d}:{m:02d}:{s:02d}.{ms:03d}"`

    3. `export_to_srt(transcript: dict[str, Any], *, include_speaker: bool = True) -> str` — per D-02:
       - Iterate segments, number each entry starting at 1
       - Format: `{i}\n{start} --> {end}\n{text}\n`
       - If include_speaker and seg has "speaker" key, prefix text with `{speaker}: `
       - Join entries with `\n` separator
       - Return empty string for empty segments

    4. `export_to_vtt(transcript: dict[str, Any], *, include_speaker: bool = True) -> str`:
       - Start with `"WEBVTT\n\n"` header (per pitfall 2)
       - Same structure as SRT but using VTT timestamp format (period separator)
       - Join entries with `\n` separator

    5. `export_to_obsidian(transcript: dict[str, Any]) -> str` — per D-06:
       - Build YAML frontmatter with `---` delimiters containing:
         title (quoted), date (YYYY-MM-DD from created_at), duration (int seconds),
         languages (comma-separated in brackets), tags (comma-separated in brackets), source
       - Body: `# {title}\n\n` then metadata lines, then AI results (summary, keywords) if present, then `---\n` separator, then segments with `[MM:SS]` timestamps (reuse existing `_format_timestamp`)

    6. `obsidian_filename(transcript: dict[str, Any]) -> str` — per D-07:
       - Extract created_at date (YYYY-MM-DD) and title
       - Sanitize title: remove `/ \ | # ^ [ ]` and other filesystem-unsafe chars, replace spaces with `_`, limit length
       - Return `f"{date}_{safe_title}.md"`

    Update module docstring to include new formats.
