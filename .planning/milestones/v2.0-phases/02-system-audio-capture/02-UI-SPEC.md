## Component Inventory

### 1. SystemAudioToggle (new widget)

**Location:** Control bar (`_control_bar`), positioned between `_duration_label` and `_record_btn` in `btn_row`.

**Layout:** Horizontal: `[duration_label] [system_audio_toggle] [record_btn] [status_label]`

**Visual spec:**
- Custom QWidget, 44px wide x 24px tall
- Track: 44x24px rounded rectangle, border-radius 12px
- Thumb: 20x20px circle, 2px inset from track edges
- OFF state: track = `background.control` (#3A3A3C / #E5E5EA), thumb = `text.primary` (#F5F5F7 / #1D1D1F)
- ON state: track = `status.recording` (#FF453A / #FF3B30), thumb = white (#FFFFFF)
- DISABLED state: track = `background.control` at 50% opacity, thumb = `text.muted` (#48484A / #AEAEB2)
- Transition: thumb slides left/right with 150ms ease-in-out animation (QPropertyAnimation)
- Cursor: PointingHandCursor when enabled, ForbiddenCursor when disabled

**Label:** "System Audio" caption label (11px, `text.secondary`) positioned 4px below the toggle, center-aligned. When recording dual-source, label color changes to `text.accent`.

**Interaction states:**
- Enabled + OFF: Click toggles ON. Aggregate Device becomes audio input.
- Enabled + ON: Click toggles OFF. Reverts to mic-only input.
- Disabled (BlackHole not installed): Click opens BlackHoleSetupWizard. Toggle does NOT change state.
- Disabled during active recording: Toggle is non-interactive (cannot change source mid-recording).

**Tooltip:**
- Enabled: "Capture system audio alongside microphone"
- Disabled (no BlackHole): "BlackHole audio driver required -- click to set up"
- Disabled (recording): "Cannot change audio source while recording"

### 2. DualLevelMeter (new widget, replaces single QProgressBar)

**Decision:** Dual stacked level meters (not badge). This gives clear real-time feedback for both audio sources.

**Location:** Replaces existing `_level_bar` QProgressBar in `_control_bar`.

**Visual spec:**
- Container: QWidget, full width of control bar, 12px total height
- Mic meter: QProgressBar, full width, 4px height, top position
  - Track: `background.control`
  - Chunk: `status.recording` (#FF453A / #FF3B30)
- System meter: QProgressBar, full width, 4px height, bottom position
  - Track: `background.control`
  - Chunk: `status.processing` (#FF9F0A / #FF9500)
- Spacing between meters: 4px (xs token)
- Labels (optional, caption size): "MIC" and "SYS" left-aligned, 11px, `text.tertiary`, only visible when dual-source recording is active

**Behavior:**
- Mic-only recording: Only mic meter visible (4px, same as current). System meter hidden. Total height = 4px (no layout shift).
- Dual-source recording: Both meters visible. Total height = 12px. Control bar height increases from 72px to 80px.
- Idle: Both meters at 0, only mic meter visible.

### 3. BlackHoleSetupWizard (new QDialog)

**Pattern:** Reuses OnboardingWizard QStackedWidget + Back/Next button pattern.

**Window spec:**
- QDialog, 500x480px, fixed size
- Title: "Scribe -- System Audio Setup"
- Modal: True (blocks main window)
- Window flags: No help button (same as OnboardingWizard)

**Pages (4 steps):**

#### Step 1: Introduction
- Heading (17px, semibold): "Capture System Audio"
- Body (14px): "To transcribe the other side of calls, Scribe needs BlackHole -- a free virtual audio driver."
- Illustration area: 200x120px centered box with `background.elevated` fill, showing a simple diagram (two audio waves merging). Implementation: QPainter line art, not an image file.
- Next button label: "Get Started"

#### Step 2: Install BlackHole
- Heading: "Install BlackHole"
- Two install options, vertical stack:
  - Option A (recommended): "Install via Homebrew" -- bordered card (`border.default`, border-radius `sm`), 14px body text with monospace command: `brew install blackhole-2ch`. Copy button (QPushButton, caption size) right-aligned in card. Label: "Copy Command"
  - Option B: "Download from GitHub" -- bordered card, body text + link-styled QPushButton: "Open Download Page". Opens `https://github.com/ExistentialAudio/BlackHole/releases` in default browser.
- Status indicator below options: QLabel with dynamic text
  - Before detection: "Waiting for BlackHole installation..." (text.tertiary, 11px)
  - After detection: checkmark prefix + "BlackHole detected!" (status.success, 11px, semibold)
- Detection: QTimer polling every 2 seconds, calling `list_audio_devices()` and checking for "BlackHole" in device names
- Next button: Disabled until BlackHole detected. Label: "Continue"

#### Step 3: Create Aggregate Device
- Heading: "Set Up Audio Mixing"
- Body: "Scribe will create an Aggregate Device that combines your microphone and BlackHole into a single input."
- Device name preview: bordered card showing device name "Scribe Audio (Mic + System)" in monospace, 14px
- Action button (centered, accent background): "Create Aggregate Device"
  - On click: Calls pyobjc CoreAudio API to create aggregate device
  - Loading state: Button text changes to "Creating...", disabled, QProgressBar indeterminate below
  - Success: Checkmark + "Aggregate Device created!" (status.success)
  - Error: Error icon + error message (status.error) + "Retry" button
- Next button: Disabled until aggregate device created. Label: "Continue"

#### Step 4: Complete
- Heading: "All Set!"
- Body: "System audio capture is ready. Toggle 'System Audio' next to the record button to capture both sides of your calls."
- Success icon: Large green checkmark, 48x48px, QPainter drawn circle + checkmark
- Finish button label: "Done"
- On finish: Dialog closes, system audio toggle becomes enabled, settings updated

**Bottom navigation:**
- Back button: Left-aligned, visible from step 2 onward. Label: "Back"
- Next/Finish button: Right-aligned. Label changes per step.
- Step indicator: "Step N of 4" caption text, center-aligned between Back and Next

### 4. Settings Dialog Extension

**Location:** New section in existing Audio tab (or new "System Audio" tab if Audio tab is crowded).

**Elements:**
- Section heading (14px, semibold): "System Audio"
- Status row: "BlackHole:" + status label
  - Installed: "Installed" (status.success color) + "Reconfigure" link-button
  - Not installed: "Not installed" (text.tertiary) + "Set Up" button (opens wizard)
- Aggregate Device row: "Device:" + device name label (text.secondary) or "Not configured" (text.tertiary)

## Error and Degradation States

| Scenario | Visual Response | Copy |
|----------|----------------|------|
| BlackHole not installed | Toggle disabled (grayed), click opens wizard | Tooltip: "BlackHole audio driver required -- click to set up" |
| BlackHole removed after setup | Toggle reverts to disabled, status bar warning | "BlackHole audio driver not found. System audio capture disabled." |
| System audio stream fails mid-recording | System level meter freezes then hides, mic meter continues | Status bar: "System audio disconnected -- continuing with microphone only" |
| Aggregate Device creation fails | Wizard step 3 shows error + retry button | "Failed to create device: {error}. Please try again." |
| BlackHole installed but aggregate not created | Toggle enabled but recording uses BlackHole directly (fallback) | Status bar: "Recording: System Audio (BlackHole)" |

New QSS rules needed in `ThemeEngine.generate_qss()`:

   - `QProgressBar#system_level_bar::chunk` uses `status.processing` instead of `status.recording`

   - Padding: `spacing.md` (16px)
